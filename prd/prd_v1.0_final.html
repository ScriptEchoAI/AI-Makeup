  • 付费、订阅、商业化

产品愿景 vs 当前交付（对齐预期）

愿景（目标态）：基于识别到的五官与肤质特征，由 AI 自动匹配眉形、唇色、眼妆结构、高光与阴影等，使预览自然、美观且个体适配；可结合云端大模型或专业美妆知识图谱迭代。

当前 app/ v1.x（已实现）：端上 MediaPipe Face Landmarker + 本地规则引擎——脸型（圆/长/方/心/椭圆）、肤色取样（深浅/冷暖启发式）、眼型几何（圆/杏/细长等），再与场合、关系组合映射到妆容包（唇、腮红、修容、眉、高光）并跟脸绘制；建议区含结构化文案与 pill。此为可解释的演示级配妆，非深度学习意义上的「全脸智能美妆成片」。

路线图（草案）：v1.x 持续打磨规则与叠层自然度、弱网与失败回退；v2 引入更强的人脸属性分析（肤色、立体度、眼型等）与更丰富妆容模板；v3+ 可选云端推理 / 品牌色卡对接 / 用户反馈闭环。以 PRD 与立项评审为准。

一、需求背景

用户在旅行、年会、节假日等出镜场景前，需要快速判断「某类妆容是否适合自己」；在帮长辈化妆时缺少可执行的结构化提示。小红书、抖音等教程无法回答「是否适合我」。本产品以实时 AR 预览 + 风格与结构建议为 MVP，区别于「事后修图」类应用。

二、需求目标

目标类型描述衡量指标目标值
效率更快完成妆容决策并获得可执行细节。首访至「建议+预览」完成时长、任务完成率。典型路径 ≤10 分钟。
体验避免强滤镜感；长辈举机场景可理解提示。主观满意度、滤镜感负反馈率。MVP 为轻度 AR + 结构建议。
信任默认本地、默认不上云。隐私相关客诉。云端能力须单独勾选。

三、用户与使用场景

用户：以年轻女性为主，覆盖多年龄段；年龄以自选与估计结合，不一致时以自选为准。场景：旅游拍照、年会、节假日；关系场景：帮长辈化妆时长辈可举着手机，需距离与光线提示。

冷启动：首次必须选择场合、年龄段、关系（给自己 / 帮长辈）。

七、异常与边界处理

源文件见 flowcharts/exception_and_edge.mmd。

flowchart TD
  A[请求相机权限] --> B{用户授权}
  B -->|拒绝| C[使用占位画面与示意妆效]
  B -->|允许| D[显示实时预览并叠层]
  C --> E[仍可查看结构与风格建议]
  D --> F{人脸检测}
  F -->|未检测到| G[提示调整距离与光线]
  F -->|检测到| H[正常预览与建议]
  G --> F
  I[帮长辈模式] --> J[显示举机距离与光线提示]
  K[年龄自选与估计不一致] --> L[以用户自选为准并弱提示]
  • 断网：MVP 默认本地处理；若未来有云，需离线降级与队列策略。
  • 空状态：未检测到人脸时持续提示调整取景。
  • 权限：拒绝相机后仍保留建议与示意叠层，引导设置中开启。
